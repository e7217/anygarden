## Situation

OpenHands와의 비교 학습에서, 설정 API가 시크릿을 반환할 때 손수 짠 마스킹에 의존하면 미래의 새 엔드포인트/모델이 실수로 평문을 흘릴 "구조적 여지"가 있다는 가설(#527)이 나왔다. OpenHands는 `SecretStr` + 직렬화-시점 레댁션으로 이를 타입 불변식화한다.

## Task

- anygarden cluster의 시크릿 반환 경로를 감사해 실제 유출 여지가 있는지 확인한다.
- 있으면 구조적으로 막고, 없으면 안전 상태를 회귀 테스트로 고정한다(YAGNI — 이미 안전한 코드를 불필요하게 리팩터하지 않는다).

## Action

감사 결과, 코드는 **이미 안전·DRY·일관적**이었다:
- `api/v1/llm_gateway.py`의 `SecretOut`은 평문 필드가 없고 비가역 `value_preview`만 반환한다.
- 마스킹은 `_mask_secret` **단일 함수**로 일원화되어 3개 시크릿 응답 경로에서 일관 사용된다.
- `api/v1/mcp_templates.py`는 주석(L106)에 명시적으로 decrypted env 값을 list 응답에 echo하지 않는다.
- 클러스터 전반에 마스킹 로직 중복/불일치는 없었다.

따라서 큰 리팩터 대신 **불변식을 회귀 테스트로 고정**하는 비례 대응을 택했다. `packages/cluster/tests/test_secret_redaction.py` 신설:
- `_mask_secret` 계약(짧은 값 → `***`, 긴 값 → 접두 12자 + `…` + 말미 4자, 중간부 은닉, 비가역).
- `SecretOut` 직렬화가 평문(및 식별 가능한 중간부)을 절대 담지 않고 프리뷰만 노출하며, 애초에 원문을 담을 필드가 스키마에 없음을 단언.

## Result

- 테스트 4개 통과(메인 venv 의존성 + worktree 코드로 검증).
- 소스 변경 없음 — 코드가 이미 불변식을 만족함을 확인하고, 그 계약을 테스트로 durable하게 고정.
- 향후 시크릿 반환 모델/엔드포인트가 추가될 때 이 테스트가 평문 유출 회귀를 잡는다.
- 후속(선택): 로깅/에러 응답까지 포함한 더 넓은 시크릿 위생 감사는 별도 범위로 분리 가능.
